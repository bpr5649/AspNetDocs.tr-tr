## <a name="what-is-distributed-caching"></a><span data-ttu-id="51223-112">Dağıtılmış önbelleğe alma nedir?</span><span class="sxs-lookup"><span data-stu-id="51223-112">What is distributed caching</span></span>

<span data-ttu-id="51223-113">Önbellek, verileri bellekte depolayarak sık erişilen uygulama verilerine yüksek aktarım hızı ve düşük gecikmeli erişim sağlar.</span><span class="sxs-lookup"><span data-stu-id="51223-113">A cache provides high throughput, low-latency access to commonly accessed application data, by storing the data in memory.</span></span> <span data-ttu-id="51223-114">Bir bulut uygulaması için en faydalı tür önbellek dağıtılmış önbelleğidir. ## <a name="when-to-use-distributed-caching"></a><span data-ttu-id="51223-120">Dağıtılmış önbelleğe alma ne zaman kullanılır?</span><span class="sxs-lookup"><span data-stu-id="51223-120">When to use distributed caching</span></span>

<span data-ttu-id="51223-121">Önbelleğe alma, verilerin yazılmasına kıyasla daha fazla okuyan uygulama iş yükleri için ve veri modeli, verileri depolamak ve önbellekte veri almak için kullandığınız anahtar/değer organizasyonunu desteklediğinde en iyi şekilde kullanılır.</span><span class="sxs-lookup"><span data-stu-id="51223-121">Caching works best for application workloads that do more reading than writing of data, and when the data model supports the key/value organization that you use to store and retrieve data in cache.</span></span> <span data-ttu-id="51223-122">Uygulama kullanıcıları çok sayıda ortak veriyi paylaştığında de daha yararlı olur; Örneğin, her kullanıcı genellikle bu kullanıcıya özgü verileri alıyorsa, önbellek pek çok avantaj sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="51223-122">It's also more useful when application users share a lot of common data; for example, cache would not provide as many benefits if each user typically retrieves data unique to that user.</span></span> <span data-ttu-id="51223-123">Verilerin sık değiştirilmediği ve tüm müşterilerin aynı verilere bakdığı için, önbelleğe almanın çok yararlı olduğu bir örnek bir ürün kataloğudur.</span><span class="sxs-lookup"><span data-stu-id="51223-123">An example where caching could be very beneficial is a product catalog, because the data does not change frequently, and all customers are looking at the same data.</span></span>

<span data-ttu-id="51223-124">Önbelleğe almanın avantajı, sürekli olarak daha fazla bir uygulama ölçeklendirilirken, kalıcı veri deposunun aktarım hızı ve gecikme gecikmeleri, genel uygulama performansı açısından daha fazla sınıra neden olur.</span><span class="sxs-lookup"><span data-stu-id="51223-124">The benefit of caching becomes increasingly measurable the more an application scales, as the throughput limits and latency delays of the persistent data store become more of a limit on overall application performance.</span></span> <span data-ttu-id="51223-125">Ancak, performansı daha da farklı nedenlerle önbelleğe alma işlemini uygulayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51223-125">However, you might implement caching for other reasons than performance as well.</span></span> <span data-ttu-id="51223-126">Kullanıcıya gösterildiğinde tam olarak güncel olması gereken veriler için, önbellek erişimi, kalıcı veri deposunun yanıt vermediği veya kullanılamadığı durumlarda bir devre kesici işlevi görebilir.</span><span class="sxs-lookup"><span data-stu-id="51223-126">For data that doesn't have to be perfectly up-to-date when shown to the user, cache access can serve as a circuit breaker for when the persistent data store is unresponsive or unavailable.</span></span>

## <a name="popular-cache-population-strategies"></a><span data-ttu-id="51223-127">Popüler önbellek popülasyon stratejileri</span><span class="sxs-lookup"><span data-stu-id="51223-127">Popular cache population strategies</span></span>

<span data-ttu-id="51223-128">Önbellekten veri alabilmek için öncelikle bu dosyayı depolamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="51223-128">In order to be able to retrieve data from cache, you have to store it there first.</span></span> <span data-ttu-id="51223-129">Bir önbellekte ihtiyacınız olan verileri almak için birkaç strateji vardır:</span><span class="sxs-lookup"><span data-stu-id="51223-129">There are several strategies for getting data that you need into a cache:</span></span>

- <span data-ttu-id="51223-130">Isteğe bağlı/önbelleğe alma</span><span class="sxs-lookup"><span data-stu-id="51223-130">On Demand / Cache Aside</span></span>

    <span data-ttu-id="51223-131">Uygulama, verileri önbellekten almaya çalışır ve önbellekte veri olmadığında ("isabetsizlik"), uygulama verileri önbellekte depolar, böylece bir sonraki sefer kullanılabilir hale gelir.</span><span class="sxs-lookup"><span data-stu-id="51223-131">The application tries to retrieve data from cache, and when the cache doesn't have the data (a "miss"), the application stores the data in the cache so that it will be available the next time.</span></span> <span data-ttu-id="51223-132">Uygulamanın bir dahaki sefer aynı verileri almaya çalıştığında, önbellekte ne aradığını bulur ("isabet").</span><span class="sxs-lookup"><span data-stu-id="51223-132">The next time the application tries to get the same data, it finds what it's looking for in the cache (a "hit").</span></span> <span data-ttu-id="51223-133">Veritabanında değiştirilen önbelleğe alınmış verileri getirmeyi engellemek için, veri deposunda değişiklik yaparken önbelleği geçersiz kılın.</span><span class="sxs-lookup"><span data-stu-id="51223-133">To prevent fetching cached data that has changed on the database, you invalidate the cache when making changes to the data store.</span></span>
- <span data-ttu-id="51223-134">Arka planda veri gönderme</span><span class="sxs-lookup"><span data-stu-id="51223-134">Background Data Push</span></span>

    <span data-ttu-id="51223-135">Arka plan hizmetleri, verileri düzenli bir zamanlamaya göre önbelleğe gönderir ve uygulama her zaman önbellekten çeker.</span><span class="sxs-lookup"><span data-stu-id="51223-135">Background services push data into the cache on a regular schedule, and the app always pulls from the cache.</span></span> <span data-ttu-id="51223-136">Bu yaklaşım, her zaman en son verileri döndürmemenizi gerektirmeyen yüksek gecikmeli veri kaynaklarıyla harika bir şekilde çalışmaktadır.</span><span class="sxs-lookup"><span data-stu-id="51223-136">This approach works great with high latency data sources that don't require you always return the latest data.</span></span>
- <span data-ttu-id="51223-137">Devre Kesici</span><span class="sxs-lookup"><span data-stu-id="51223-137">Circuit Breaker</span></span>

    <span data-ttu-id="51223-138">Uygulama normalde kalıcı veri deposuyla doğrudan iletişim kurar, ancak kalıcı veri deposunda kullanılabilirlik sorunları olduğunda, uygulama verileri önbellekten alır.</span><span class="sxs-lookup"><span data-stu-id="51223-138">The application normally communicates directly with the persistent data store, but when the persistent data store has availability problems, the application retrieves data from cache.</span></span> <span data-ttu-id="51223-139">Veriler önbelleğe alınmış ya da arka planda veri gönderme stratejisi kullanılarak önbelleğe alınmış olabilir.</span><span class="sxs-lookup"><span data-stu-id="51223-139">Data may have been put in cache using either the cache aside or background data push strategy.</span></span> <span data-ttu-id="51223-140">Bu, performans geliştirme stratejisi yerine bir hata işleme stratejisidir.</span><span class="sxs-lookup"><span data-stu-id="51223-140">This is a fault handling strategy rather than a performance enhancing strategy.</span></span>

<span data-ttu-id="51223-141">Verileri önbellekte tutmak için, uygulamanız verileri oluşturduğunda, güncelleştirdiğinde veya silerse ilgili önbellek girdilerini silebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51223-141">In order to keep data in the cache current, you can delete related cache entries when your application creates, updates, or deletes data.</span></span> <span data-ttu-id="51223-142">Uygulamanızın bazı durumlarda biraz zaman güncel olmayan verileri almasını istiyorsanız, eski önbellek verilerinin nasıl olabildiğinden ilgili bir sınır ayarlamak için yapılandırılabilir bir zaman aşımı süresine güvenebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51223-142">If it's alright for your application to sometimes get data that is slightly out-of-date, you can rely on a configurable expiration time to set a limit on how old cache data can be.</span></span>

<span data-ttu-id="51223-143">Mutlak süre sonu (önbellek öğesi oluşturulduktan bu yana geçen süre) veya kayan süre sonu (önbellek öğesine en son erişildiği zamandan bu yana geçen süre) yapılandırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="51223-143">You can configure absolute expiration (amount of time since the cache item was created) or sliding expiration (amount of time since the last time a cache item was accessed).</span></span> <span data-ttu-id="51223-144">Verilerin çok eski hale gelmesini engellemek için önbellek süre sonu mekanizmasına bağlı olduğunuzda mutlak süre sonu kullanılır.</span><span class="sxs-lookup"><span data-stu-id="51223-144">Absolute expiration is used when you are depending on the cache expiration mechanism to prevent the data from becoming too stale.</span></span> <span data-ttu-id="51223-145">BT uygulamasını düzelttikten sonra eski önbellek öğelerini el ile çıkaracağız ve en güncel verileri önbellekte tutmak için kayan süre sonu kullanacağız.</span><span class="sxs-lookup"><span data-stu-id="51223-145">In the Fix It app, we'll manually evict stale cache items and we'll use sliding expiration to keep the most current data in cache.</span></span> <span data-ttu-id="51223-146">Seçtiğiniz süre sonu ilkesinden bağımsız olarak, önbelleğin bellek sınırına ulaşıldığında önbelleğin en eski (en son kullanılan veya LRU) öğelerini otomatik olarak çıkaracaktır.</span><span class="sxs-lookup"><span data-stu-id="51223-146">Regardless of the expiration policy you choose, the cache will automatically evict the oldest (Least Recently Used or LRU) items when the cache's memory limit is reached.</span></span>
