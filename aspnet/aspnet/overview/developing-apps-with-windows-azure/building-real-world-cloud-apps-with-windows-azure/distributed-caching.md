## <a name="when-to-use-distributed-caching"></a><span data-ttu-id="f2733-120">Ne zaman kullanılacağı dağıtılmış önbelleğe alma</span><span class="sxs-lookup"><span data-stu-id="f2733-120">When to use distributed caching</span></span>

<span data-ttu-id="f2733-121">En iyi, daha fazla okuma verilerin yazılmasını daha yapın ve ne zaman önbelleğindeki verileri depolama ve alma için kullandığınız anahtar/değer kuruluş veri modelini destekler uygulama iş yükleri için önbelleğe alma çalışır.</span><span class="sxs-lookup"><span data-stu-id="f2733-121">Caching works best for application workloads that do more reading than writing of data, and when the data model supports the key/value organization that you use to store and retrieve data in cache.</span></span> <span data-ttu-id="f2733-122">Uygulama kullanıcılarına ortak verilerin çok paylaştığınızda de daha kullanışlıdır; Örneğin, her kullanıcı genellikle bu kullanıcı için benzersiz bir veri alıyorsa önbellek gibi birçok avantaj sağlamaz.</span><span class="sxs-lookup"><span data-stu-id="f2733-122">It's also more useful when application users share a lot of common data; for example, cache would not provide as many benefits if each user typically retrieves data unique to that user.</span></span> <span data-ttu-id="f2733-123">Örneği burada önbelleğe alma çok yararlı olabilir bir ürün kataloğu, verileri sık değişmeyen ve tüm müşterilerine aynı verilere baktığımızda olmasıdır.</span><span class="sxs-lookup"><span data-stu-id="f2733-123">An example where caching could be very beneficial is a product catalog, because the data does not change frequently, and all customers are looking at the same data.</span></span>

<span data-ttu-id="f2733-124">Önbelleğe alma avantajı daha fazla uygulama ölçeklenen, aktarım hızı sınırlarını ve gecikme süresi gecikmeler kalıcı veri deposu daha genel uygulama performansını sınırlı hale geldikçe giderek ölçülebilir olur.</span><span class="sxs-lookup"><span data-stu-id="f2733-124">The benefit of caching becomes increasingly measurable the more an application scales, as the throughput limits and latency delays of the persistent data store become more of a limit on overall application performance.</span></span> <span data-ttu-id="f2733-125">Ancak, başka bir nedenle performans de daha önbelleğe alma uygulayabilir.</span><span class="sxs-lookup"><span data-stu-id="f2733-125">However, you might implement caching for other reasons than performance as well.</span></span> <span data-ttu-id="f2733-126">Kalıcı veri deposu yanıt vermiyor veya kullanılabilir olduğunda kullanıcıya gösterilen mükemmel güncel olması gerekmez, veriler için önbellek erişim için bir devre kesici görebilir.</span><span class="sxs-lookup"><span data-stu-id="f2733-126">For data that doesn't have to be perfectly up-to-date when shown to the user, cache access can serve as a circuit breaker for when the persistent data store is unresponsive or unavailable.</span></span>

## <a name="popular-cache-population-strategies"></a><span data-ttu-id="f2733-127">Popüler önbellek popülasyonu stratejileri</span><span class="sxs-lookup"><span data-stu-id="f2733-127">Popular cache population strategies</span></span>

<span data-ttu-id="f2733-128">Verileri önbellekten oluşturabilmek, ilk var. depolamak zorunda.</span><span class="sxs-lookup"><span data-stu-id="f2733-128">In order to be able to retrieve data from cache, you have to store it there first.</span></span> <span data-ttu-id="f2733-129">İhtiyacınız olan verilerin bir önbelleğe alma için birçok strateji vardır:</span><span class="sxs-lookup"><span data-stu-id="f2733-129">There are several strategies for getting data that you need into a cache:</span></span>

- <span data-ttu-id="f2733-130">İsteğe bağlı olarak / edilgen önbellek</span><span class="sxs-lookup"><span data-stu-id="f2733-130">On Demand / Cache Aside</span></span>

    <span data-ttu-id="f2733-131">Önbellekten veri almak uygulama çalışır ve önbellek verilerini (bir "miss") sahip olmadığında, sonraki kullanılabilir olacaktır, böylece uygulama verileri önbellekte depolar.</span><span class="sxs-lookup"><span data-stu-id="f2733-131">The application tries to retrieve data from cache, and when the cache doesn't have the data (a "miss"), the application stores the data in the cache so that it will be available the next time.</span></span> <span data-ttu-id="f2733-132">Uygulama aynı verileri almaya çalıştığında ne, önbellekte ("isabet") tarafından bakılıyor bulur.</span><span class="sxs-lookup"><span data-stu-id="f2733-132">The next time the application tries to get the same data, it finds what it's looking for in the cache (a "hit").</span></span> <span data-ttu-id="f2733-133">Veritabanı üzerinde değişmiş önbelleğe alınmış verileri getirme önlemek için veri deposuna değişiklikler yaparken önbelleği geçersiz.</span><span class="sxs-lookup"><span data-stu-id="f2733-133">To prevent fetching cached data that has changed on the database, you invalidate the cache when making changes to the data store.</span></span>
- <span data-ttu-id="f2733-134">Arka plan veri gönderimi</span><span class="sxs-lookup"><span data-stu-id="f2733-134">Background Data Push</span></span>

    <span data-ttu-id="f2733-135">Arka plan Hizmetleri verileri önbelleğe düzenli bir zamanlamaya ve uygulama her zaman çeken önbellekten gönderin.</span><span class="sxs-lookup"><span data-stu-id="f2733-135">Background services push data into the cache on a regular schedule, and the app always pulls from the cache.</span></span> <span data-ttu-id="f2733-136">Bu yaklaşım çalıştığı harika her zaman gerektirmeyen yüksek gecikme veri kaynaklarıyla en son verileri döndürür.</span><span class="sxs-lookup"><span data-stu-id="f2733-136">This approach works great with high latency data sources that don't require you always return the latest data.</span></span>

    <span data-ttu-id="f2733-138">Uygulamayı, normalde kalıcı veri deposuyla doğrudan iletişim kurar, ancak kalıcı veri deposu kullanılabilirlik sorunları olduğunda, uygulama önbellekten veri alır.</span><span class="sxs-lookup"><span data-stu-id="f2733-138">The application normally communicates directly with the persistent data store, but when the persistent data store has availability problems, the application retrieves data from cache.</span></span> <span data-ttu-id="f2733-139">Veri önbelleği kenara veya arka plan veri itme stratejisi kullanarak önbellekte konmuş olabilir.</span><span class="sxs-lookup"><span data-stu-id="f2733-139">Data may have been put in cache using either the cache aside or background data push strategy.</span></span> <span data-ttu-id="f2733-140">Bir performans geliştirme stratejisi yerine stratejisi işleme bir hata budur.</span><span class="sxs-lookup"><span data-stu-id="f2733-140">This is a fault handling strategy rather than a performance enhancing strategy.</span></span>

<span data-ttu-id="f2733-141">Önbellekte verileri güncel tutmak için uygulamanızı oluşturur, güncelleştirmeleri veya verileri sildiğinde ilgili önbellek girişi silebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="f2733-141">In order to keep data in the cache current, you can delete related cache entries when your application creates, updates, or deletes data.</span></span> <span data-ttu-id="f2733-142">Bazı durumlarda biraz güncel olmayan verileri almak uygulamanız için bunu tamamdır ise, verilerin ne kadar eski önbellek sınırlamak için yapılandırılabilir sona erme zamanı güvenebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="f2733-142">If it's alright for your application to sometimes get data that is slightly out-of-date, you can rely on a configurable expiration time to set a limit on how old cache data can be.</span></span>

<span data-ttu-id="f2733-143">Mutlak zaman aşımı (önbellek öğesinin oluşturulduğu zaman miktarı) veya kayan zaman aşımı (bir önbellek öğeye erişildiği son kez daraltılmasından miktarı) yapılandırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="f2733-143">You can configure absolute expiration (amount of time since the cache item was created) or sliding expiration (amount of time since the last time a cache item was accessed).</span></span> <span data-ttu-id="f2733-144">Mutlak zaman aşımı, verileri çok eski duruma gelmesini önlemek için Önbellek süre sonu mekanizması bağlı olduğunda kullanılır.</span><span class="sxs-lookup"><span data-stu-id="f2733-144">Absolute expiration is used when you are depending on the cache expiration mechanism to prevent the data from becoming too stale.</span></span> <span data-ttu-id="f2733-145">Uygulamada Düzelt, biz eski önbellek öğeleri el ile Tahliye ve olmaadığını önbellekte en güncel verileri tutmak için kullanacağız.</span><span class="sxs-lookup"><span data-stu-id="f2733-145">In the Fix It app, we'll manually evict stale cache items and we'll use sliding expiration to keep the most current data in cache.</span></span> <span data-ttu-id="f2733-146">Seçtiğiniz süre sonu ilkesi bağımsız olarak, önbellek önbelleğin bellek sınırına ulaşıldığında en eski (en son kullanılan veya LRU) öğeleri otomatik olarak çıkarırsınız.</span><span class="sxs-lookup"><span data-stu-id="f2733-146">Regardless of the expiration policy you choose, the cache will automatically evict the oldest (Least Recently Used or LRU) items when the cache's memory limit is reached.</span></span>
